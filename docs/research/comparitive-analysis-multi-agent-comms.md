## Evaluation frame and what ÔÇťstructured communicationÔÇŁ means here

This comparison treats ÔÇťstructured communication and notationÔÇŁ as a spectrum of techniques that reduce ambiguity (and therefore failure) when language models exchange information with software, tools, or other models. The five systems you listed fall into three overlapping families:

## Cross-cutting patterns and trade-offs

A useful way to interpret the five systems is to ask where they ÔÇťpush structure enforcementÔÇŁ:

BAML and TypeChat push structure into **schema + (robust parsing/repair)**. BAML leans on a language-level contract and parsing strategy that can tolerate flexible model output (e.g., extra text around JSON). ţłÇciteţłéturn2view1ţłü TypeChat leans on explicit validation against the TypeScript schema and, if validation fails, iterative repair using compiler diagnostics. ţłÇciteţłéturn5view0ţłéturn8view2ţłü These are excellent for ÔÇťapp integrationÔÇŁ workflows where you need concrete objects and can accept some retry cost.

LMQL pushes structure into **decode-time constraints**, preventing invalid continuations token-by-token and failing early when constraints canÔÇÖt be met. ţłÇciteţłéturn3view0ţłü The trade-off is that decode-time control depends on backend capabilities and tends to be easiest on local models or APIs that expose the right hooks (hence LMQLÔÇÖs explicit backend list). ţłÇciteţłéturn13view0ţłü

DSPy pushes structure into **program-level modularity and optimisation**: rather than trying to make one prompt perfectly constrained, it frames the pipeline as code + signatures + metrics, and then compiles/optimises the prompts (and potentially weights) to satisfy the objective. ţłÇciteţłéturn4view1ţłéturn10view0ţłü This is powerful when your main failure mode is ÔÇťpipeline doesnÔÇÖt work reliably across domains/modelsÔÇŁ rather than ÔÇťJSON sometimes invalidÔÇŁ.

OCTAVE pushes structure into **durable artefacts**: canonical forms, embedded schema, transformation logs, and explicit compression/loss declarations. ţłÇciteţłéturn12view1ţłéturn12view3ţłü ItÔÇÖs a different answer to ÔÇťstructured communicationÔÇŁ because its unit is not a one-off function call but a document that moves across agents and time.

One particularly important cross-cutting point: even the strongest structure enforcement typically guarantees the **shape** of output, not its **semantic correctness**. Research on constrained decoding notes that many uses ÔÇťguarantee constraint compliance given a schemaÔÇŁ, but also highlights limited understanding of effectiveness in practice and evaluates ÔÇťquality of generated outputsÔÇŁ separately from compliance. ţłÇciteţłéturn15view0ţłü This applies broadly: you can force valid JSON that is still wrong, irrelevant, or hallucinated.

## Gaps that remain unaddressed across all five

Even taken together, these five systems do not fully close several recurring gaps in LLM and multi-agent communication.
